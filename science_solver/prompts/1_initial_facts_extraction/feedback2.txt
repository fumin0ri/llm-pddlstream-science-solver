## Checklist
1:All predicates in ## Predicates must follow the format (predicateName ?variableName)
2:Make sure that the definitions and values ​​correspond and that each element of ('params', ...) in ## Initial Facts is separated by a comma.
3:Is the problem analysis appropriate? All phenomena mentioned in the problem statement must be considered.
4:PDDL does not allow non-ASCII characters such as Greek letters (λ,Δ,ρ,θ,σ) and must be written using only alphanumeric characters in ## Predicates. You cannot use non-ASCII characters for either predicates or variable names. In the case of λ, using 'lambda' would make it a Python reserved word, so use something like 'lam' instead.
5:All variables in each predicate in ## Initial Facts must be numeric. They must not contain any characters.
6:You cannot assign different values to the same predicate. Instead of writing something like (dist ?name ?num) with facts such as ('dist', 1, 2.0), ('dist', 2, 3.5) you have to write (dist_a ?num_a), (dist_b ?num_b), etc.
7:For every predicate defined in ## Predicates, you must assign numerical values in ## Initial Facts. In other words, the predicates listed under ## Predicates must correspond exactly to those listed under ## Initial Facts.
8:Do not use the same variable name across different predicates. For example, (mass_a ?m) and (mass_b ?m) are not allowed; you must use (mass_a ?m_a) and (mass_b ?m_b) instead.
9:You must extract only the numerical values provided in ##domain and problem. No values that require calculation may be included. 
10:If no specific numerical values are given in the domain and the problem, there is no need to define predicates. If there are no predicates to define, write empty code blocks for ## Predicates and ## Initial Facts.
11:In the Predicates section, you cannot use a single predicate like (dist ?d) and then have facts such as ('dist', 50) and ('dist', 100) in the Initial Facts section. Instead, you need to distinguish the predicates, for example by using (dist_a ?d_a) and (dist_b ?d_b), with corresponding facts ('dist_a', 50) and ('dist_b', 100).
12:Variable names that differ only in uppercase and lowercase cannot be used across predicates. For example, (mass_a ?m) and (mass_b ?M) will cause an error, so you need to use something like (mass_a ?ma) or (mass_b ?mb) instead.
# Example

## Domain and problem

Our AI agent is a computational robot that solves specific problems that require calculations and returns an answer, which must also indicate the process it followed, such as what mathematical formulas it applied.

A small car of mass 800 kg starts from rest and accelerates uniformly to a speed of 20 m/s in 10 seconds on a flat road. During this time, the engine provides a constant power output, and the resistive force acting on the car is 400 N. What is the power output of the engine? (Ignore air resistance other than the 400 N resistive force.)

```

(velocity ?v) ;v[m/s]
(time ?t) ;t[s]
(resistive_force ?fr) ;fr[N]
```

## Initial Facts
Here we write the numerical values ​​corresponding to the predicates we defined earlier. Note that each element must be separated by a comma.
```

('velocity', 20.0) 
('time', 10.0) 
('resistive_force', 400.0) 
```
