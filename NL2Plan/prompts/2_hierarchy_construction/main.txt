# Examples
## Example 1
### Domain and Problem
I've got a building with five floors and two elevators, and I want you to plan the elevator trips for Bob and Sally to both go to floor 5 in a climate-friendly way.

### Types
The types are:
- floor: Different levels in the building, numbered 1 to 5.
- elevator: Elevators that can transport people between floors.
- person: Individuals like Bob and Sally who need to travel to different floors.
- building: The building we're modeling.

### Reasoning
#### Type Usages
- *floor*: floors represent the different levels within the building that both *elevator* and *person* can access.
- *elevator*: Elevators are devices that move between floors, transporting persons. They will be at floors and sometimes contain persons.
- *person*: Bob and Sally will travel between floors using the elevators, meaning they can be present at *locations* or inside elevators.
- *building*: The building contains floors, elevators, and persons. it will not be used for any action or predicate in particular.

#### Usage Analysis
- *floor* is the only type where elevators and persons will be at, no other type shares this property.
- elevators and persons will both move between different floors. To model this joint usage, we will add a *movable* type. Other than that, their usage is unique since elevators is the only type which moves directly between floors and persons are the only type which can enter elevators.
- *building* will remain at the top level since it has no joint usage with any other type.

### Hierarchy
```
- object: Object is always root, everything is an object
    - floor: Different levels in the building, numbered 1 to 5.
    - movable: Types that can move between floors.
        - elevator: Elevators that can transport people between floors.
        - person: Individuals like Bob and Sally who need to travel between floors.
    - building: The building that contains floors and elevators.
```

## Example 2
### Domain and Problem
In this domain, the AI agent is a wildlife conservation management system. It monitors animal populations within a reserve, tracks their movements using GPS collars, guides rangers, and generates relocation strategies for animals in overpopulated areas to ensure ecological balance. Movement can occur only between neighboring locations, and animals can only be in habitat locations. Furthermore, rangers can escort animals between habitats. Rangers can also guard locations where they are.

There are four habitats currently, one centered and three around it. The surrounding ones contain our deer and rabbits. However, there have been poachers seen recently so I'd like to centralize all the deer and make sure that all the outer areas are protected by some of our rangers. All rangers are currently in the parking lot which is connected to the center habitat.

### Types
The types are:
- rabbit: Small jumping animals we wish to safeguard.
- habitat: Areas within the reserve that provide specific conditions for animals.
- ranger: Workers employed in the conservatory that can move animals and guard locations.
- deer: Large gracing animals we wish to safeguard.
- location: A more general term for different areas within the reserve.
- population: A group of animals.

### Reasoning
#### Type Usages
- *rabbit*: rabbits will move between habitats, escorted by rangers, and are restricted to being only in habitats.
- *habitat*: habitats are specific areas where animals can live, though rangers can also move between these.
- *ranger*: rangers will be used to escort animals and guard locations, which means they need to be able to move between locations, similar to animals.
- *deer*: *deer* will move between habitats, escorted by rangers, and are restricted to being only in habitats.
- *location*: This defines general areas where rangers can be.
- *population*: This models a group of various animals, however there is no obvious means as to how it will be used. Likely it will be connected using predicates to specific animals.

#### Usage Analysis
- *rabbit*, *deer*, and *ranger* all move between locations and/or habitats. Since they share this commonality, we will introduce a new meta type called *movable* to capture this behavior. All of these will be subtypes of *movable*.
- *rabbit* and *deer* are both limited to only moving between habitats and being able to be escorted by rangers. To model this joint behavior we will add a new type called *animal* with *rabbit* and *deer* as subtypes.
- *habitat* is a more specific form of *location*, both *ranger* and the new *animal* can move between habitats but only rangers can be in locations. So, *habitat* will be modeled as a subtype of *location*.
- *population* does not share any specific behaviors with other types and will remain at the top level. While it relates to animals and groups thereof, it is not a metatype of animals or anything of the like (as can be seen through the usage.)

### Hierarchy
```
- object: Object is always root, everything is an object
    - movable: Types that can move between locations.
        - animal: Animals that can move between habitats within the reserve.
            - rabbit: A specific type of animal we want to safeguard.
            - deer: A specific type of animal we want to safeguard.
        - ranger: Workers who can move between locations, escort animals, and guard locations.
    - location: Types that can be visited by movable entities.
        - habitat: Areas within the reserve that provide specific conditions for animals.
    - population: A group of animals within the reserve.
```

## Example 3
### Domain and Problem
The AI agent is a scheduling system for a house construction company with administrators who can create orders, workers who perform them, and vehicles to transport things.

### Types
The types are:
- administrator: Workers who can create orders for other workers.
- wall: The exterior of a house. Requires a floor to be built first.
- construction_worker: People who can perform a variety of tasks if there is an order for it.
- vehicle: Used to transport workers and material.
- house: What is constructed.
- floor: The bottom of a house. Can be built wherever.
- depot: Where material is stored and vehicles can be loaded.
- roof: The top of a house. Requires the walls to be built first.
- location: Places where workers and machines can be.
- mansion: A large type of house.
- library: A building filled with books.

### Reasoning
#### Type Usages
- *administrator*: Administrators will be used to create orders. They could also possibly move around between locations or similar via vehicles.
- *wall*: Walls will be used to model the status of the building.
- *construction_worker*: Construction workers will be used to create walls, floors, and roofs. They will have to move to the locations via vehicles to build there.
- *vehicle*: Vehicles will be used to transport workers and material. They can be built at houses.
- *house*: Houses will be used as a type of location that can have walls, floors, and roofs constructed on it.
- *floor*: Floors will be used to model the status of houses. They can be built at houses.
- *depot*: Depots are a type of location where material is stored and can be loaded onto vehicles.
- *roof*: Roofs will be used to model the status of houses. They can be built at houses.
- *location*: Locations will be used to model all the places where workers and vehicles can be at.
- *mansion*: Mansions will be used to model a specific type of house with extra requirements.
- *library*: Libraries will be used to model a specific type of house with extra requirements.

#### Usage Analysis
- *house*, *library*, and *mansion* have common usage as they all model houses to be built. Since *library* and *mansion* are more specific forms of *house* these will be the subtypes of *house*.
- *administrator*, *construction_worker*, and *vehicle* will all be used to model objects able to move. As such, we will add a new meta type called *movable* for all types that can move between locations and make *administrator*, *construction_worker*, and *vehicle* a subtype of *movable*.
- *wall*, *floor*, and *roof* are all objects which can be built at houses. However, none of them are more specified versions of the others. As such, we will add a new meta type called *house_component* consisting of all types that can be built by a construction worker at a house given an order, with *roof*, *wall*, and *floor* as subtypes of *house_component*.
- *house* (and its subtypes *mansion* and *library*), *depot*, and *location* all model places which can be visited. As *house* and *depot* are more specific versions of *location*, *location* will be used as the meta type for both *depot* and *house*.
- *administrator* and *construction_worker* are both workers which have joint behavior such as transportation via vehicles. To model this, we'll add a joint meta type called *worker*.

### Hierarchy
Combining the above reasoning, we reach the following set of types with the following descriptions.

```
- object: Object is always root, everything is an object
    - movable: Types that are able to move between locations.
        - worker: Employees of the company who move through the use of vehicles.
            - administrator: Administrators can move between locations and create orders.
            - construction_worker: Construction workers can move between locations and execute orders.
        - vehicle: Vehicles can move between locations and transport material.
    - house_component: Types that can be built by construction workers at a house, given an order for it.
        - floor: Floor can be built at a house by construction workers given an order.
        - wall: Walls can be built at a house by construction workers given an order and that the floor is built.
        - roof: Roof can be built at a house by construction workers given an order and that the walls are built.
    - location: Types that can be visited by the "movable" types.
        - house: Houses can be visited, and constructed with house components.
            - mansion: Mansions can be visited, and constructed with house components.
            - library: Libraries can be visited, and constructed with house components.
        - depot: Depots can be visited and contain material that vehicles can be loaded with.
```
