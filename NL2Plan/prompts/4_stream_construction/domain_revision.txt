You can use the following advanced PDDL syntax if needed:
- Grounded Predicates: `(predicate1)` is a predicate without any arguments. It can be used to model general properties, such as plan status.
- Equality: `(= ?o1 ?o2)` only evaluates to true if ?o1 and ?o2 are the same object.
- Disjunctive  Preconditions: `(or (predicate1 ?t) (predicate2 ?t))` evaluates to true if at least one of "(predicate1 ?t)" or "(predicate2 ?t)" evaluates to true. Only applicable in preconditions.
- Existential Preconditions: `(exists (?t - type) (predicate ?t))` evaluates to true if there is any object ?t of type "type" such that "(predicate ?t)" evaluates to true. Only applicable in preconditions.
- Implication: `(imply (predicate1 ?t) (predicate2 ?t))` evaluates to true if either "(predicate1 ?t)" is false or both are true. This is the closest equivalent to an if-statement in PDDL. Only applicable in preconditions.
- Universal Preconditions: `(forall (?t - type) (predicate ?t))` evaluates to true if "(predicate ?t)" evaluates to true for all objects ?t of type "type". Only applicable in preconditions. Can also be used to set sets of predicates in effects.
- Conditional Effects: `(when (ANTECENT) (CONSEQUENCE))`. The effects in CONSEQUENCE are only applied if the ANTECENT expression evaluates to true. Only applicable in effects. Note that if there are several conditional effects, even if they're disjoint, they need to be enveloped in an "and" statement.
